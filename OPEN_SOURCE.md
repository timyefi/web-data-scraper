# 关于本开源版本

你可能注意到，本项目的 `scripts/`、`references/`、`assets/` 目录均未包含在开源版本中。这是有意为之的设计。

## 我们的初衷

我们希望通过这个开源项目，分享一种 **AI 辅助数据采集的思路与范式**：

- 如何让 AI 像人一样操作网站，同时像程序一样批量获取数据
- 如何在"分析师的操作思路"和"程序员的编程技巧"之间找到结合点
- 如何通过 API 优先 + 浏览器兜底的混合策略，高效且合规地完成信息采集

这些思路在 README 和 SKILL.md 中有完整的展示。我们希望大家从这个开源版本中主要获得的，正是这样一种认知和方法论层面的启发。

## 为什么只开放工作流定义

AI 辅助投研是一个复杂的系统工程。在实际的专业场景中，数据采集需要紧密结合具体的工作流来定义——不同的研究目标、数据来源、输出要求都会影响整个 Skill 的设计和执行。

正因如此，我们只展示了 Skill 整个工作流的定义框架（SKILL.md），而未提供具体的实现细节，包括：

- **`scripts/`**：网站分析、API 发现、数据提取、批量下载等脚手架代码
- **`references/`**：各网站的详细操作指南、反检测策略、分页处理方案等参考文档
- **`assets/`**：新网站的配置模板

这些内容需要根据实际业务场景进行定制化配置，脱离具体工作流直接使用的意义有限。
