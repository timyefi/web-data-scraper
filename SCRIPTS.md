# 关于脚手架代码

你可能注意到本项目的 `scripts/` 目录并未包含在开源版本中。这是有意为之的设计。

## 原因

本 Skill 的定位并非一个开箱即用的通用工具，而是一套**面向专业投研场景的方法论与知识体系**。

考虑到卖方研究服务的商业模式特点：

- **专业服务导向**：数据采集的真正价值不在于工具本身，而在于对目标网站的理解、查询条件的构建、异常情况的处理等专业知识。这些需要结合具体业务场景来交付。
- **持续维护成本**：各网站接口和反爬策略频繁变动，脚手架代码需要持续跟进适配，不适合作为独立工具分发。
- **质量保障**：由团队提供专业服务可以确保数据采集的准确性和完整性，避免因不当使用导致的问题。

## 你仍然可以获得的

本开源版本完整提供了：

- **方法论文档**（`references/`）：网站分类策略、元素识别技巧、分页处理方案、反检测指南
- **网站操作指南**：巨潮资讯、中国货币网、国家统计局、政府采购网等具体站点的详细操作方法
- **配置模板**（`assets/`）：新网站的标准化配置模板
- **完整工作流定义**（`SKILL.md`）：三阶段工作流的完整描述

具备编程能力的团队可以根据这些文档，结合自身技术栈实现相应的脚手架代码。如需完整的脚本代码，建议联系团队获取。
